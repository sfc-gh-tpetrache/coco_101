# Pipeline Overview

## Goal
Analyze support tickets in Snowflake by classifying them into product categories, issue types, and priority, then enriching them with summaries and insights for PMs and Support Managers.

## Flow

1. `RAW_SUPPORT_TICKETS`
   - Raw input data

2. `DT_TICKET_CLASSIFICATION` (Dynamic Table)
   - Uses `AI_CLASSIFY`
   - Outputs:
     - product_category
     - issue_type
     - priority_bucket

3. `DT_TICKET_ENRICHMENT` (Dynamic Table)
   - Uses `AI_COMPLETE`
   - Inputs: raw ticket + classification outputs
   - Outputs:
     - summary_text
     - rationale
     - manager_recommendation

## Refresh
- Dynamic Tables use `REFRESH_MODE = INCREMENTAL` — only new/changed rows are processed on subsequent refreshes
- Data should be **at least daily fresh**
- Shorter `target_lag` can be used for demos

## Cost Monitoring
- Track AI function credit consumption via `SNOWFLAKE.ACCOUNT_USAGE.CORTEX_AI_FUNCTIONS_USAGE_HISTORY`
- View has up to 60 min latency; running queries update every ~30 min


## Rules
- Use `AI_CLASSIFY` only for labels
- Use `AI_COMPLETE` only for per-row summaries and rationales
- Use `AI_AGG` for cross-row insights — it handles datasets larger than a single LLM context window
- Do not classify the same field twice
- Keep raw data separate from derived outputs

## Validation
- Row counts match across stages
- Labels stay within taxonomy
- Key fields (summary, rationale) are populated
- Spot check sample records

## Notebook Role
The notebook is the main demo:
- Query dynamic tables directly
- Validate outputs
- Show sample records
- Build charts

## Key Outputs

### PM
- Tickets by product_category
- Issue trends over time

### Support Manager
- Tickets by priority_bucket
- High-priority (P0/P1) trends

### AI-Aggregated Insights (AI_AGG)
- **Support Manager perspective** — what's breaking, escalation patterns, recurring issues, Monday-morning action items
- **Product Manager perspective** — feature requests, competitor signals, sprint prioritization evidence, cross-module issues
- Each persona receives top 3 actionable insights ranked by impact, generated by `AI_AGG` over `DT_TICKET_ENRICHMENT`
